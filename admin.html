Hosting on Framer

Option 1  Recommended: Subdomain on Vercel

Host the graph at graph.camdenshuman.com via Vercel (free). Framer handles your main site; Vercel handles the graph. You link between them normally — no conflicts, no Framer limitations.

Steps: Push knowledge-graph.html, admin.html, and graph-data.js to a GitHub repo. Connect to Vercel. In Vercel → Settings → Domains, add graph.camdenshuman.com. In your Framer site settings add the DNS record Vercel provides. Done — the graph is live at its own URL.

To update: export graph-data.js from this admin, commit to GitHub, Vercel auto-deploys in ~30 seconds.

Option 2  Embed in Framer via iframe

In Framer, add an Embed component and point it to your graph URL. The full interactive graph renders inside your Framer page. Visitors never leave your site.

Tradeoff: the graph is constrained to the embed block dimensions. Best for embedding alongside other content. Use Option 1 if you want the graph to be a full-screen destination.

Framer steps: Insert → Embed → set URL to https://graph.camdenshuman.com → width 100%, height 700px (or set to viewport height for full-screen feel).

⚡ Fastest path to live (~45 minutes)

1. Create a GitHub account if needed. 2. New repo, upload 3 files. 3. vercel.com → New Project → import repo → deploy. 4. Vercel → Settings → Domains → add graph.camdenshuman.com. 5. Framer site settings → add the DNS record Vercel gives you. The graph is live.
